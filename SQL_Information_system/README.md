## 一个基于Qt和SQLite的员工信息管理系统，提供直观的用户界面和强大的数据管理功能。

## 项目概述

本项目是一个使用Qt框架开发的员工信息管理系统，主要功能包括员工数据的增删改查、图片管理、数据过滤等。系统采用SQLite作为数据库，提供了友好的图形用户界面，方便用户进行数据管理操作。

## 功能特性

- **数据库管理**：支持SQLite数据库的打开、关闭和管理
- **员工信息管理**：添加、编辑、删除员工信息
- **数据显示**：表格形式展示员工数据，支持双击编辑
- **图片管理**：支持员工照片的上传、存储和显示
- **数据过滤**：按性别等条件过滤员工数据
- **自定义编辑器**：为特定字段（如性别、部门、省份）提供下拉选择功能
- **数据验证**：确保数据的完整性和一致性

## 技术栈

- **框架**：Qt 6.9.2
- **编译器**：MinGW 64-bit
- **数据库**：SQLite
- **开发环境**：Qt Creator

## 核心功能

### 1. 数据库操作

- **打开数据库**：通过文件对话框选择SQLite数据库文件
- **数据模型**：使用`QSqlTableModel`管理数据库表数据
- **数据提交**：通过`submitAll()`提交数据更改到数据库

### 2. 员工信息管理

- **添加员工**：自动生成工号，初始化基本信息
- **编辑员工**：双击表格单元格进行编辑，或通过右侧信息面板编辑
- **删除员工**：支持删除选中的员工记录
- **员工照片**：支持存储和显示员工照片，自动调整大小以适应显示区域

### 3. 自定义编辑器

- **性别选择**：下拉选择男/女
- **部门选择**：下拉选择部门
- **省份选择**：下拉选择省份

### 4. 数据过滤

- **按性别过滤**：通过单选按钮选择过滤条件
- **重置过滤**：清除过滤条件，显示所有数据

## 安装与运行

### 环境要求

- Qt 6.9.2或更高版本
- MinGW 64-bit编译器
- SQLite数据库

### 编译步骤

1. 打开Qt Creator
2. 选择"文件" -> "打开文件或项目"
3. 选择项目目录中的`SQL_Information_system.pro`文件
4. 配置构建套件（选择MinGW 64-bit）
5. 点击"构建"按钮编译项目
6. 运行编译后的可执行文件

### 运行方式

1. 启动应用程序
2. 点击"打开"按钮选择SQLite数据库文件
3. 在表格中查看和编辑员工信息
4. 使用右侧面板查看和修改详细信息
5. 使用过滤功能按条件筛选数据

## 使用指南

### 基本操作

- **打开数据库**：点击工具栏中的"打开"按钮，选择SQLite数据库文件
- **添加员工**：点击"添加"按钮，系统会自动生成工号并初始化基本信息
- **编辑员工**：双击表格单元格直接编辑，或在右侧面板修改后点击"保存"按钮
- **删除员工**：选中要删除的员工记录，点击"删除"按钮
- **保存更改**：点击"保存"按钮将更改提交到数据库
- **取消更改**：点击"取消"按钮放弃未保存的更改

### 照片管理

- **上传照片**：在右侧面板中点击照片区域，选择本地图片文件
- **照片显示**：系统会自动调整照片大小以适应显示区域
- **照片存储**：照片以二进制形式存储在数据库中

### 数据过滤

- **性别过滤**：选择"男"或"女"单选按钮，系统会过滤显示相应性别的员工
- **显示全部**：选择"全部"单选按钮，显示所有员工数据

## 技术实现

### 数据模型与视图

- 使用`QSqlTableModel`作为数据模型，与SQLite数据库交互
- 使用`QTableView`展示数据，支持表格形式的编辑
- 使用`QDataWidgetMapper`将数据模型与UI组件绑定

### 自定义委托

- 继承`QStyledItemDelegate`实现自定义的下拉框编辑器
- 为特定列（性别、部门、省份）设置自定义委托，提供下拉选择功能

### 信号与槽

- 使用Qt的信号槽机制处理用户操作和数据变化
- 实现了编辑、保存、取消等操作的信号处理

### 内存管理

- 使用Qt的父对象机制管理对象生命周期，避免内存泄漏
- 正确处理数据库连接的打开和关闭

## 注意事项

- 系统默认使用SQLite数据库，数据库文件应保存在`SQLite`目录中
- 照片以二进制形式存储，可能会增加数据库文件大小
- 编辑数据时，确保在点击"保存"按钮前完成所有编辑操作
- 系统支持基本的数据验证，但请确保输入数据的有效性
